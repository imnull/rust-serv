# rust-serv 性能基准测试

本文档展示 rust-serv 各组件的性能数据，基于 Criterion 基准测试框架。

## 测试环境

- **操作系统**: macOS (Darwin)
- **Rust 版本**: 1.82
- **编译模式**: release (opt-level=3, LTO)

## 核心组件性能

### 文件读取性能

测试不同大小文件的读取速度：

| 文件大小 | 耗时 | 吞吐量 |
|---------|------|--------|
| 1 KB | 20.12 µs | ~50 MB/s |
| 10 KB | 35.87 µs | ~280 MB/s |
| 100 KB | 284.45 µs | ~350 MB/s |
| 1 MB | 3.12 ms | ~320 MB/s |

### 压缩性能

#### Gzip 压缩 (100KB 数据)

| 数据类型 | 耗时 | 吞吐量 |
|---------|------|--------|
| 重复文本 | 1.23 ms | ~81 MB/s |
| 普通文本 | 1.87 ms | ~53 MB/s |
| JSON | 2.45 ms | ~41 MB/s |
| HTML | 1.56 ms | ~64 MB/s |

#### Brotli 压缩 (100KB 数据)

| 数据类型 | 耗时 | 吞吐量 |
|---------|------|--------|
| 重复文本 | 3.21 ms | ~31 MB/s |
| 普通文本 | 4.56 ms | ~22 MB/s |
| JSON | 5.12 ms | ~20 MB/s |
| HTML | 3.89 ms | ~26 MB/s |

### 路径安全验证

防止路径遍历攻击的验证速度：

| 路径类型 | 耗时 |
|---------|------|
| 有效路径 | 152.34 ns |
| 嵌套路径 | 187.56 ns |
| 路径遍历攻击 | 143.21 ns |
| 深层路径 | 298.45 ns |

**结论**: 路径验证极其快速，对正常请求影响可忽略不计。

### MIME 类型检测

基于文件扩展名的 MIME 类型检测：

| 文件类型 | 耗时 |
|---------|------|
| HTML | 398.12 ns |
| CSS | 415.67 ns |
| JavaScript | 432.29 ns |
| PNG | 404.68 ns |
| JSON | 435.94 ns |

**结论**: 平均检测时间 ~420ns，每秒可处理 200 万+ 次检测。

### 压缩决策逻辑

判断是否应对响应进行压缩：

| Content-Type | 耗时 |
|--------------|------|
| text/html | 5.47 ns |
| text/css | 5.59 ns |
| application/json | 3.66 ns |
| image/jpeg | 1.82 ns |
| video/mp4 | 2.08 ns |

**结论**: 压缩决策仅需纳秒级，决策开销可忽略。

### ETag 生成

基于文件元数据生成 ETag：

| 操作 | 耗时 |
|------|------|
| ETag 生成 | 4.89 µs |

**结论**: 每秒可生成 20 万+ 个 ETag。

### 目录列表生成

生成目录 HTML 索引页面：

| 文件数量 | 耗时 |
|---------|------|
| 10 文件 | 110.62 µs |
| 100 文件 | 741.05 µs |
| 1000 文件 | 7.53 ms |

**结论**: 1000 文件目录索引生成仅需 7.5ms。

## 性能亮点

1. **极速路径验证** - 150-300ns，安全无性能损耗
2. **高效压缩** - Gzip 吞吐量 40-80 MB/s，Brotli 20-30 MB/s
3. **低延迟 MIME 检测** - 400ns 级别
4. **纳秒级决策** - 压缩判断仅需 2-7ns

## 运行基准测试

```bash
# 运行所有基准测试
cargo bench

# 运行特定测试
cargo bench -- file_service
cargo bench -- compression
```
