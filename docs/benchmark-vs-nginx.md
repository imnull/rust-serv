# rust-serv vs nginx 性能对比

## 测试环境

- **操作系统**: macOS Darwin
- **CPU**: 多核
- **nginx 版本**: 1.29.5
- **rust-serv 版本**: 0.3.0
- **测试工具**: oha v1.14.0
- **测试参数**: 10,000 请求，100/500 并发

## 测试结果

### 1. 小文件 HTML (38 bytes)

| 指标 | nginx | rust-serv | 提升 |
|------|-------|-----------|------|
| **QPS** | 17,467 | 55,818 | **3.2x** |
| 平均延迟 | 5.66 ms | 1.74 ms | 3.3x |
| P99 延迟 | 6.67 ms | 3.45 ms | 1.9x |
| 成功率 | 100% | 100% | - |

### 2. 小文件文本 (10 bytes)

| 指标 | nginx | rust-serv | 提升 |
|------|-------|-----------|------|
| **QPS** | 18,911 | 55,453 | **2.9x** |
| 平均延迟 | 5.18 ms | 1.72 ms | 3.0x |

### 3. 高并发测试 (500 并发)

| 指标 | nginx | rust-serv | 提升 |
|------|-------|-----------|------|
| **QPS** | 17,340 | 50,817 | **2.9x** |
| 平均延迟 | 27.61 ms | 9.17 ms | 3.0x |
| 成功率 | 100% | 100% | - |

## 性能总结

| 场景 | rust-serv 优势 |
|------|---------------|
| 静态 HTML | **3.2x 更快** |
| 纯文本文件 | **2.9x 更快** |
| 高并发 | **2.9x 更快** |
| 延迟 | **3x 更低** |

## 关键发现

1. **rust-serv 在静态文件服务上显著快于 nginx**
   - 小文件场景下 QPS 提升 3 倍
   - 延迟降低约 70%

2. **高并发场景表现优异**
   - 500 并发下保持 100% 成功率
   - QPS 稳定在 50,000+

3. **响应时间分布更集中**
   - rust-serv P99 延迟仅为 nginx 的 50%
   - 延迟波动更小，用户体验更稳定

## 测试方法

```bash
# 安装测试工具
cargo install oha

# 启动服务器
nginx -c /path/to/nginx.conf
./rust-serv

# 运行基准测试
oha -n 10000 -c 100 http://localhost:8080/index.html
```

## 注意事项

- 测试在同一台机器上进行，生产环境结果可能有差异
- nginx 使用默认配置，可通过调优提升性能
- rust-serv 目前对小文本文件优化更好
