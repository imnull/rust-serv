# Rust HTTP 静态服务器 - 需求文档

## 项目概述

本项目旨在创建一个高性能、安全、可配置的 Rust HTTP 静态文件服务器，采用测试驱动开发（TDD）范式进行渐进式开发。

## 功能需求

### MVP (最小可行产品) 功能

1. **基础 HTTP 服务**
   - 支持 HTTP/1.1 协议
   - 监听指定端口（默认 8080）
   - 返回静态文件内容

2. **静态文件服务**
   - 从指定根目录提供文件
   - 支持常见文件类型（HTML, CSS, JavaScript, 图片等）
   - 正确设置 Content-Type 响应头
   - 返回 200 OK（文件存在）或 404 Not Found（文件不存在）

3. **目录索引**
   - 当请求目录时返回 HTML 文件列表
   - 显示文件名、大小、修改时间
   - 隐藏隐藏文件（以 `.` 开头的文件）

4. **路径安全**
   - 防止路径遍历攻击（`..`, `/etc/passwd` 等）
   - 阻止访问服务根目录之外的文件
   - 返回 403 Forbidden 对非法路径

### 增强功能

5. **范围请求支持**
   - 支持 HTTP Range 头
   - 返回 206 Partial Content
   - 正确设置 Content-Range 头
   - 支持视频/音频流媒体

6. **配置系统**
   - TOML 格式配置文件
   - 可配置端口、根目录、日志级别
   - 支持环境变量覆盖
   - 提供默认配置

7. **日志和监控**
   - 记录所有请求（访问日志）
   - 记录错误和异常
   - 支持不同日志级别（ERROR, WARN, INFO, DEBUG）
   - 结构化日志输出

8. **内容压缩**
   - 支持 Gzip 压缩
   - 支持 Brotli 压缩
   - 根据客户端 Accept-Encoding 自动选择
   - 不压缩已压缩文件（如 .png, .jpg）

9. **缓存控制**
   - 生成 ETag
   - 支持 If-None-Match / If-Match
   - 返回 304 Not Modified
   - 可配置缓存策略

10. **TLS 支持**
    - 支持 HTTPS
    - 加载 TLS 证书和密钥
    - 可选的 HTTP 到 HTTPS 重定向

### 可选功能

11. **CORS 支持**
    - 可配置 CORS 头
    - 支持预检请求

12. **速率限制**
    - 防止 DDoS
    - 基于 IP 的限制

13. **认证**
    - Basic Auth
    - Token 认证

14. **虚拟主机**
    - 多域名支持
    - 不同域名映射不同目录

## 非功能需求

### 性能

- 支持 1000+ 并发连接
- 吞吐量：> 10,000 requests/second（简单静态文件）
- 响应时间：P99 < 100ms（小文件）
- 内存占用：稳定在合理范围内（< 100MB）

### 安全性

- 防止路径遍历攻击
- 防止目录列出攻击
- 防止文件包含攻击
- 支持 TLS 加密
- 可配置的访问控制
- 输入验证和净化

### 可靠性

- 99.9% 可用性
- 优雅的错误处理
- 优雅的服务器关闭
- 日志持久化

### 可维护性

- 清晰的代码结构
- > 80% 测试覆盖率
- 完整的文档
- 类型安全（Rust 保证）
- 模块化设计

### 可扩展性

- 插件式中间件架构
- 易于添加新功能
- 配置驱动

### 可配置性

- 通过配置文件定制所有行为
- 支持环境变量
- 运行时热重载（可选）

## 技术约束

- 使用 Rust 语言
- 使用 Tokio 异步运行时
- 使用 Hyper HTTP 服务器框架
- 支持 Linux 和 macOS
- 最小 Rust 版本：1.70+

## 用户体验需求

- 清晰的错误消息
- 友好的配置文件
- 命令行参数支持
- 帮助文档

## 兼容性需求

- 兼容 HTTP/1.1
- 兼容主流浏览器
- 支持 UTF-8 文件名
- 遵循 RFC 标准（HTTP, MIME 等）

## 测试需求

- 单元测试覆盖率 > 80%
- 集成测试覆盖所有主要功能
- 性能基准测试
- 安全测试

## 文档需求

- 快速开始指南
- 配置文档
- API 文档
- 架构文档
- 贡献指南

## MVP 定义

**MVP 包含以下功能**：
1. 基础 HTTP 服务
2. 静态文件服务（支持常见 MIME 类型）
3. 目录索引
4. 路径安全
5. 配置系统（TOML）
6. 基础日志

**MVP 不包含**：
- TLS 支持
- 压缩
- 缓存
- 范围请求
- CORS
- 认证

这些功能将在 MVP 后续迭代中逐步添加。

## 验收标准

项目完成需满足：
1. 所有 MVP 功能已实现
2. 所有测试通过（单元测试 + 集成测试）
3. 文档完整且准确
4. 代码通过 clippy 检查
5. 性能达到或超过非功能需求
6. 安全性审查通过
