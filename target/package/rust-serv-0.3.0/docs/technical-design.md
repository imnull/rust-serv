## 技术栈选择

### 核心运行时

**Tokio**
- 选择理由：成熟、性能优异、社区活跃
- 优势：
  - 基于事件循环的高性能异步运行时
  - 丰富的生态系统
  - 完善的定时器和任务调度
  - 跨平台支持

### HTTP 框架

**Hyper + Tower**
- Hyper：
  - 底层 HTTP 库，提供细粒度控制
  - 性能优异
  - HTTP/1.1 和 HTTP/2 支持
  - 类型安全

- Tower：
  - 中间件抽象
  - 服务模式
  - 可组合的处理链

### 配置处理

**Serde + TOML**
- Serde：序列化/反序列化框架
- TOML：易读易写的配置格式
- 优势：类型安全、编译时验证

### 日志

**Tracing**
- 选择理由：结构化日志、异步友好
- 功能：
  - 结构化日志输出
  - Span 上下文跟踪
  - 多种输出格式
  - 可配置日志级别

### 压缩

- **flate2**: Gzip 压缩
- **brotli**: Brotli 压缩
- 选择标准：根据客户端 Accept-Encoding 自动选择

### MIME 类型

**mime_guess**
- 基于文件扩展名推断 MIME 类型
- 内置常见类型映射

### 测试

- **内置测试框架**：单元测试
- **reqwest**：集成测试（HTTP 客户端）

## 数据流

### 请求处理流程

```
1. 客户端发送 HTTP 请求
   │
   ▼
2. Hyper 接收请求并解析
   │
   ▼
3. Logging 中间件记录请求
   │
   ▼
4. Compression 中间件检查 Accept-Encoding
   │
   ▼
5. Cache 中间件检查 ETag/缓存
   │
   ▼
6. Handler 路由请求
   │
   ▼
7. Path Security 验证路径
   │
   ▼
8. File Service 读取文件/目录
   │
   ▼
9. MIME Detector 检测类型
   │
   ▼
10. Response Builder 构建响应
    │
    ▼
11. Compression 中间件压缩内容（如需要）
    │
    ▼
12. Cache 中间件添加缓存头
    │
    ▼
13. Logging 中间件记录响应
    │
    ▼
14. Hyper 发送响应给客户端
```

```
1. 命令行参数

   │
   ▼

   │
   ▼

   │

   ▼
```

## 错误处理

### 错误类型

```rust
pub enum Error {
    // 配置错误
    Config(ConfigError),

    // IO 错误
    Io(io::Error),

    // HTTP 错误
    Http(HttpError),

    // 路径安全错误
    PathSecurity(PathSecurityError),

    // 文件未找到
    NotFound,

    // 权限拒绝
    Forbidden,

    // 内部错误
    Internal(String),
}
```

### 错误处理策略

1. **优雅降级**：遇到错误时返回用户友好的消息
2. **日志记录**：记录所有错误上下文
3. **错误链**：保留原始错误信息
4. **客户端友好**：对暴露的错误信息进行过滤

### HTTP 状态码映射

- `200 OK`: 成功
- `206 Partial Content`: 范围请求
- `304 Not Modified`: 缓存命中
- `400 Bad Request`: 无效请求
- `403 Forbidden`: 路径安全违规
- `404 Not Found`: 文件不存在
- `416 Range Not Satisfiable`: 无效范围
- `500 Internal Server Error`: 服务器错误
- `503 Service Unavailable`: 过载

## 安全设计

### 路径安全

1. **路径规范化**：解析 `..` 和符号链接
2. **路径验证**：确保路径在根目录内
3. **权限检查**：验证文件访问权限

### 输入验证

1. **路径字符过滤**：阻止特殊字符
2. **文件名长度限制**
3. **请求大小限制**

### HTTP 安全头

```
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Strict-Transport-Security: max-age=31536000; includeSubDomains
```

### TLS 配置

- 支持 TLS 1.2+
- 强密码套件
- 证书验证
- HSTS

## 性能优化

### 并发模型

- **Tokio 多线程调度器**：充分利用多核
- **异步 I/O**：非阻塞文件读取
- **连接池**：复用连接

### 内存优化

- **零拷贝**：最小化数据复制
- **流式传输**：大文件不全部加载到内存
- **缓冲区复用**

### 缓存策略

- **内存缓存**：热点文件缓存
- **ETag**: 客户端缓存
- **Last-Modified**: 条件请求

### 文件系统优化

- **sendfile**：零拷贝文件传输（Linux）
- **内存映射文件**：大文件读取

## 性能指标

### 目标指标

- **并发连接**: 10,000+
- **请求/秒**: 50,000+（小文件）
- **内存占用**: < 100MB
- **启动时间**: < 1s

### 优化技巧

1. **零拷贝传输**：使用 hyper::Body 的包装器
2. **连接复用**：保持 HTTP/1.1 Keep-Alive
3. **内存预分配**：合理设置缓冲区大小
4. **线程池调优**：根据核心数设置
